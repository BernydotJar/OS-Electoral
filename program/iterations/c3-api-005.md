## Acceptance criteria

1. `GET /api/v1/tenants/{tenant_id}/campaigns/{campaign_id}/readiness` requires authenticated, fresh server-owned tenant authorization.
2. Authorization matches exact `tenant`, `campaign`, `action=read`, `resource_type=campaign_readiness`, `resource_id`, `purpose=Assess assigned campaign readiness`, validity and revocation state before persistence is queried.
3. The projection is deterministic and limited to operational setup: campaign name, jurisdiction, stage, and at least one active workspace.
4. The response explicitly states that readiness is not political, legal, finance, security, publication, or production approval.
5. Every successful sensitive read appends an immutable tenant/campaign audit event in the same transaction; no outbox event is emitted for the observation-only read.
6. SQLAlchemy/PostgreSQL, unavailable, and in-memory adapters implement one protocol.
7. Missing, archived, foreign-tenant, foreign-campaign, wrong-purpose, wrong-resource, revoked/absent-grant, and adapter-scope-leak cases fail closed with sanitized errors.
8. OpenAPI exposes the typed response and bearer security boundary.
9. Focused tests, full regression, PostgreSQL integration, lint, format, typecheck, program truth, safety and secret scans pass.
10. Production remains `BLOCKED`; no readiness result is treated as a human gate receipt.

## Critic findings and repairs

1. **Audit-chain ordering risk:** timestamps were initially captured before the tenant lock, which could make a later serialized append sort behind an earlier event. Repaired by issuing timestamps under a session-bound tenant lock and enforcing monotonic microsecond ordering.
2. **Shared invariant drift:** campaign and workspace writers had separate audit append implementations. Repaired by routing all three boundaries through the shared appender.
3. **Inactive coverage gate:** `fail_under=90` existed but `make test` did not invoke pytest-cov; the real value was initially `89.29%`. Added fail-closed branch tests, raised actual coverage to `90.64%`, and made the gate executable.
4. **Eval inventory gap:** required eval IDs were not machine-readable. Added an exact fail-closed catalog; absent capabilities remain `NOT_IMPLEMENTED` rather than inferred PASS.
5. **Projection-claim drift:** a custom adapter could construct internally inconsistent next actions, reason codes, workspace counts or omitted limitation codes. Repaired with whole-model invariants that derive every summary field from the canonical ordered checks.
