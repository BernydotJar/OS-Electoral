## Bounded objective

Strengthen the pre-production security and privacy baseline without enabling any external integration or environment. Protect append-only evidence from direct mutation by constrained database roles and make data classification, purpose, retention posture and prohibited political-data fields machine-verifiable.

## Acceptance criteria

1. Reversible migration adds one hardened trigger function and non-owner `UPDATE/DELETE` denial to every declared append-only journal.
2. Protected tables include audit events, durable idempotency receipts, candidate section approvals, War Room snapshots, Strategy decisions and Agent runs.
3. Trigger execution uses `session_user`, table ownership and a fixed `pg_catalog` search path; application-controlled custom settings cannot bypass it.
4. Database owner/member access remains an explicit break-glass path for reviewed migration/offboarding work and is never described as immutable external evidence.
5. PostgreSQL proves a constrained `NOSUPERUSER`/`NOBYPASSRLS` role can insert but cannot rewrite or delete audit evidence.
6. PostgreSQL introspection proves every required table owns the expected enabled trigger.
7. A versioned machine-readable data policy classifies required record types, records purposes, retention posture, deletion mode, processor boundary and legal-review state.
8. Prohibited political-data fields include individual vote intention, persuadability scoring, psychological-vulnerability targeting, biometric persuasion and covert tracking.
9. Live AI/integration processors remain disabled and secrets/tokens are prohibited from application records.
10. Policy validator fails closed on missing classes/records, unknown classifications, absent purpose/owner, unjustified production-ready retention, processor drift or removed prohibitions.
11. Threat model and privacy documentation distinguish local technical enforcement from independent privacy/security/legal acceptance.
12. CI/local verification runs the policy validator, adversarial tests, migration checks, secret/dependency/SAST and final E2E without changing protected settings or deploying.

## Non-goals

- no claim of cryptographic signing, external anchoring or absolute database immutability;
- no legal retention period or lawful-basis determination;
- no attachment, export, backup, live provider, AWS or production processor enablement;
- no branch-protection mutation, deployment, apply, spending or external campaign effect.
