## Operating Loop

Repeat until the Definition of Done is met or every remaining task is blocked by a specific unavailable authoritative input.

### Phase 0 — Resume Safely

- inspect branch, working tree, recent commits, PRs, and issue;
- detect work completed by another agent;
- read existing artifacts before overwriting them;
- maintain a task ledger using `TODO`, `IN_PROGRESS`, `PASS`, `PARTIAL`, `BLOCKED`, and `FAILED`;
- continue from the first incomplete executable task.

### Phase 1 — Acquire and Authenticate

- acquire the official cartography image from the authenticated URL or `POLITICS_ROOT`;
- record SHA-256 and image dimensions;
- confirm authority, municipality, and source identity;
- save temporary derivatives only under `.workspace/` or another ignored directory;
- update the source audit and acquisition log.

### Phase 2 — Prepare Controlled Review

- create high-resolution crops or tiles when useful;
- run OCR using Spanish and English models when available;
- use multiple segmentation modes where needed;
- preserve raw OCR separately from reviewed text;
- assign stable region identifiers to every reviewed area.

### Phase 3 — Review Labels Visually

For each candidate label:

- compare OCR against the image;
- classify label type;
- record raw OCR, reviewed text, region, confidence, and note;
- use `CONFIRMED`, `CORRECTED`, `UNRESOLVED`, or `NOT_PRESENT`;
- do not normalize ambiguous names silently.

### Phase 4 — Build Inventory

- create center rows only where the official source supports center identity;
- distinguish electoral area labels from polling-center labels;
- record missing code, address, or relationship fields explicitly;
- deduplicate only with evidence;
- evaluate the confirmed center count against the official total of 18.

### Phase 5 — Resolve JRV Assignments

- search for authoritative center-level JRV lists or ranges;
- populate assignments only from explicit official records;
- validate inclusive ranges arithmetically;
- use municipal JRV range only as a total consistency check;
- leave the assignment CSV header-only when no official assignment exists.

### Phase 6 — Control Crosswalks

- create crosswalk rows only when an official source explicitly supports the relationship;
- no name-only, proximity, or campaign-knowledge crosswalks;
- mark plausible but unsupported relationships `UNRESOLVED` in the quality report, not in the confirmed crosswalk.

### Phase 7 — Validate Immediately

After every material artifact:

- parse CSVs;
- validate required columns and enums;
- check duplicates;
- confirm every accepted row has source provenance;
- ensure confirmed center count does not exceed 18 without documented conflict;
- validate JRV arithmetic;
- scan for `/Users/`, PII, and prohibited political recommendations;
- run `git diff --check`;
- fix locally resolvable failures before continuing.

### Phase 8 — Record State

After each increment:

- update the issue with artifact, evidence, validation, blocker, and next task;
- update the implementation report;
- make a focused commit;
- push when possible;
- continue without waiting for routine approval.
